Fix It — 4 Guardrails

Each fix stacks on the previous one. Layer all four for production AI systems.

1

Source grounding

Tell the model to only use the provided document. Forces faithfulness to the source.

Answer ONLY from the provided source document.
If the answer is not in the source, say:
"I don't have enough information from the provided source."
Never use external knowledge.
→ Reduces hallucination rate by ~50%
2

Confidence labels

Ask the model to declare how supported each answer is. Makes uncertainty visible.

For every answer, output:
Answer: [your answer]
Source support: [Direct / Inferred / Not found]
Confidence: [High / Medium / Low]
Risk: [Low / Medium / High]
→ Makes hidden uncertainty visible and actionable
3

Hard refusal rule

Explicitly forbid guessing. Forces "I don't know" instead of a confident wrong answer.

Never guess, infer, or use external knowledge.
If you cannot find the answer in the provided source,
respond: "I cannot confirm this from the provided document."
→ Single highest-impact fix — closes the remaining gap
4

Self-verification chain

Ask the model to verify its own answer against the source before outputting it.

After generating your answer, re-read the source and verify:
Is every claim I made explicitly in the source?
If not, revise your answer to remove unsupported claims.
→ Catches edge cases the other rules miss
Key insight: The refusal rule (Fix 3) is the single most impactful change. It converts unsupported answers into explicit refusals — making failure visible instead of hiding it behind a confident-sounding wrong answer.
